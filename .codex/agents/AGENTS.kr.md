# 에이전트 정의 안내

이 문서는 사람이 보기 쉽게 정리한 한국어 참고본입니다.
Codex의 실제 instruction discovery에는 사용되지 않습니다.
실제 적용되는 규칙은 같은 위치의 `AGENTS.md`를 기준으로 합니다.

## 목적

- 이 디렉터리의 파일들은 Codex 서브에이전트의 역할별 동작을 정의합니다.
- 각 에이전트는 `AGENTS.md` 체인을 통해 공유 규칙을 상속받는 것을 전제로 합니다.

## 프롬프트 설계 원칙

- 각 에이전트 프롬프트는 그 역할의 고유 책임에 집중합니다.
- 기대 출력 형태, 제약, 하지 말아야 할 일을 분명하게 적습니다.
- 공통 코딩 규칙이나 일반 워크플로우 규칙을 모든 에이전트에 반복해서 복사하지 않습니다.
- `sandbox_mode`는 역할 책임에 맞게 설정합니다.
- 일반적으로 기획, 디자인, 리뷰는 `read-only`, 구현 역할만 쓰기 권한을 사용합니다.
