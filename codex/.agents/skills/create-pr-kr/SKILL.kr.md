# Create PR (KR) 참고본

이 문서는 사람이 읽기 쉽게 정리한 한국어 참고본입니다.
Codex의 실제 skill trigger와 instruction 로딩에는 사용되지 않습니다.
실제 적용되는 skill 본문은 같은 위치의 `SKILL.md`를 기준으로 합니다.

## 용도

- git 변경사항을 분석해 한국어 팀 PR 컨벤션에 맞는 커밋 제목과 PR 본문을 만드는 skill입니다.
- 커밋 제목과 PR 본문을 생성한 뒤, git/gh 명령으로 PR을 자동으로 생성하거나 기존 PR을 수정합니다.
- 한국어 커밋 요약, 한국어 제목 규칙, 체크리스트 항목은 팀 컨벤션 자체이므로 한국어로 유지합니다.

## 관련 파일

- `SKILL.md`: Codex가 실제로 읽는 skill 본문입니다.
- `agents/openai.yaml`: skill 목록 UI에서 보이는 이름, 짧은 설명, 기본 프롬프트 같은 메타데이터를 정의합니다.

즉, 실제 PR workflow는 `SKILL.md`가 담당하고,
`openai.yaml`은 사람이 skill을 고르거나 호출할 때 보조하는 표시용 설정에 가깝습니다.

## 언제 쓰나

- 한국어 팀 PR 컨벤션으로 PR을 생성하거나 수정해 달라고 요청할 때
- 변경사항을 바탕으로 한국어 커밋 제목과 PR 본문을 만들어야 할 때
- 커밋, push, PR 생성 또는 수정을 한 번에 자동으로 처리해야 할 때

## 작업 흐름

1. 현재 git diff를 분석해 변경 파일과 변경 성격을 파악합니다.
2. 브랜치명에서 이슈 번호를 추출하고, 없으면 멈추고 안내 메시지를 출력합니다.
3. 변경사항을 가장 잘 나타내는 commit type 하나를 고릅니다.
4. `<Type>(<IssueNumber>): <한글 요약>` 형식으로 커밋 제목을 만듭니다.
5. `.github/pull_request_template.md`를 원본으로 PR 본문을 채우고, 선택한 타입에 해당하는 체크리스트 한 개만 체크합니다.
6. `git add -A` → `commit` → `push` 후 기존 PR을 감지해 `gh pr edit` 또는 `gh pr create`로 처리하고 URL을 출력합니다.

## 핵심 기준

- 이슈 번호는 반드시 존재해야 하며, 없으면 진행하지 않습니다.
- 커밋 제목은 50자 이내, 마침표와 특수문자 없이, 개조식 한글로 작성합니다.
- 체크리스트는 선택한 commit type에 해당하는 항목 하나만 체크합니다.
- 명령은 순서대로 실행하고 실패 시 멈추며, 강제 push와 인터랙티브 git 플래그는 사용하지 않습니다.
- PR이 생성되거나 수정되면 URL을 출력합니다.
