# Refactor 참고본

이 문서는 사람이 읽기 쉽게 정리한 한국어 참고본입니다.
Codex의 실제 skill trigger와 instruction 로딩에는 사용되지 않습니다.
실제 적용되는 skill 본문은 같은 위치의 `SKILL.md`를 기준으로 합니다.

## 용도

- 대상 파일을 프로젝트 표준에 비추어 분석하고, 리팩토링 포인트를 제안한 뒤 승인 시에만 코드를 수정하는 skill입니다.
- 리팩토링 기준은 `codex/AGENTS.md`의 표준에서만 가져옵니다. (`Implementation Standards`, `Performance`, `TypeScript And React`, `React`, `Accessibility`, `Verification` 섹션)
- 기능 동작은 변경하지 않습니다. 구조·타입·가독성 개선에만 한정합니다.

## 관련 파일

- `SKILL.md`: Codex가 실제로 읽는 skill 본문입니다.
- `agents/openai.yaml`: skill 목록 UI에서 보이는 이름, 짧은 설명, 기본 프롬프트 같은 메타데이터를 정의합니다.

즉, 실제 리팩토링 workflow는 `SKILL.md`가 담당하고,
`openai.yaml`은 사람이 skill을 고르거나 호출할 때 보조하는 표시용 설정에 가깝습니다.

## 언제 쓰나

- 특정 파일이나 범위를 동작 변경 없이 리팩토링하고 싶을 때
- `codex/AGENTS.md` 표준에 어긋나는 부분을 정리하고 싶을 때
- 수정 전에 제안 목록을 보고 승인 여부를 직접 결정하고 싶을 때

## 작업 흐름

1. 표준 소스 없음, 지정 파일 없음, 대상·범위 미지정 상태에서는 멈춥니다.
2. `codex/AGENTS.md`를 읽어 리팩토링 기준으로 사용합니다.
3. 입력에 따라 대상을 정합니다. 파일 명시 → 해당 파일, 디렉토리 → 하위 `.ts`/`.tsx`, "전체"/미지정 → `src/` 하위 `.ts`/`.tsx`.
4. 표준 기준으로 분석한 뒤, 수정 전에 제안 목록을 먼저 보여주고 승인을 기다립니다.
5. 승인된 항목만 한 번에 하나씩 파일을 수정합니다.
6. 적용 / 대상 없음 / 취소 중 맞는 형식으로 최종 결과를 출력합니다.

## 핵심 기준

- 리팩토링 기준은 반드시 `codex/AGENTS.md`의 표준에 근거합니다. 표준에 없는 취향·스타일 선호는 제안하지 않습니다.
- 기능 동작을 바꾸지 않습니다. 구조·타입·가독성 개선에만 한정합니다.
- 승인 전에는 어떤 파일도 수정하지 않습니다.
- 승인된 항목만 수정하고, 연관 코드라도 승인되지 않은 부분은 건드리지 않습니다.
