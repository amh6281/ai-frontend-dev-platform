# 프로젝트 AGENTS 안내

이 문서는 사람이 보기 쉽게 정리한 한국어 참고본입니다.
Codex의 실제 instruction discovery에는 사용되지 않습니다.
실제 적용되는 규칙은 같은 위치의 `AGENTS.md`를 기준으로 합니다.

## 로딩 방식

- 이 작업 공간에서는 `codex/` 디렉터리 자체를 Codex 루트처럼 취급합니다.
- 따라서 기본적으로 `codex/AGENTS.md`를 기준으로 instruction discovery가 이뤄집니다.
- 같은 디렉터리에 `AGENTS.override.md`가 있으면 일반 `AGENTS.md`보다 우선합니다.
- 대체 파일명은 `.codex/config.toml`의 `project_doc_fallback_filenames` 설정을 따릅니다.

## 기본 원칙

- 이 작업 공간의 Codex 규칙은 `codex/AGENTS.md`를 기준으로 관리합니다.
- 변경 범위는 가능한 한 작업에 필요한 최소 수준으로 유지합니다.
- 동작을 바꾸기 전에 현재 코드를 먼저 확인합니다.
- 큰 리팩터링보다 방어 가능한 가장 작은 변경을 우선합니다.
- 최종 요약은 실제 변경점, 리스크, 검증 내용 중심으로 작성합니다.

## 협업

- planner, designer, frontend engineer, reviewer는 서로 다른 역할로 다룹니다.
- 역할 간 핸드오프 시 가정, 리스크, 열린 질문을 분명히 남깁니다.
- 스타일 의견보다 정확성, 회귀, 접근성, 검증 누락을 우선합니다.

## 구현 기준

- 의미 있는 하드코딩 값은 이름 있는 상수로 바꿉니다.
- 축약형보다 의도가 드러나는 이름을 선호합니다.
- 함수는 한 가지 책임에 집중하도록 작게 유지합니다.
- 반복 로직은 복사보다 추출을 우선합니다.
- 작업과 무관한 정리는 하지 않습니다.

## TypeScript 및 React

- `any`는 도입하지 않습니다.
- `unknown`, 제네릭, 판별 유니언, 타입 가드를 우선합니다.
- `React.FC`는 사용하지 않고 명시적 props 인터페이스를 사용합니다.
- `enum`보다 `as const`와 유니언 타입을 선호합니다.
- 객체 계약은 `interface`, 조합이나 유니언은 `type`을 우선합니다.
- export되는 유틸과 공용 헬퍼에는 반환 타입을 명시합니다.
- 타입 전용 import에는 `import type`을 사용합니다.
- 가드나 안전한 narrowing으로 표현 가능한 경우 무리한 assertion을 피합니다.

## 접근성

- 일반 컨테이너보다 시맨틱 엘리먼트를 선호합니다.
- 인터랙션 요소의 키보드 접근성을 유지합니다.
- 아이콘 전용 컨트롤에는 접근 가능한 이름을 제공합니다.
- 의미 있는 이미지에는 적절한 `alt`를 제공합니다.
- 색상만으로 의미를 전달하지 않습니다.

## 검증

- 변경한 정확한 동작을 검증합니다.
- 버그 수정이나 안정 동작 변경 시 테스트를 추가하거나 갱신합니다.
- 읽기 쉬우면서 경계 조건과 실패 경로를 다루는 테스트를 선호합니다.
- 실제 확인하지 않은 동작을 검증했다고 쓰지 않습니다.
